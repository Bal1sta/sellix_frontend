# Sellix — Frontend (React + Vite)

Витрина и кабинеты мультиарендной B2B2C-платформы Sellix. Подключён к Django REST
бэкенду. Фронт и бэк рассчитаны на деплой **на одном сервере за nginx**, поэтому
все запросы идут на тот же origin (`/api/v1`) — CORS не используется.

## Технологии
React 19 · React Router 7 · Vite 7. Без внешних UI-библиотек; стили — CSS-модули.

## Структура (ключевое)

```
src/
  config/runtime.js      — базовый URL API/медиа (env-управляемый)
  api/                   — HTTP-клиент и модули эндпоинтов
    client.js            — fetch + JWT + авто-refresh + нормализация ошибок
    tokenStore.js        — хранение access/refresh
    auth.js storefront.js orders.js seller.js producer.js admin.js
  auth/                  — авторизация на стороне фронта
    AuthContext.jsx      — провайдер: user, роли, вход/выход
    guards.jsx           — RequireAuth / RequireRole
    oauth.js             — редиректы Яндекс ID / VK ID (+ sandbox)
  components/
    Pages/AuthPages/     — вход, OAuth-callback, выбор роли, вход админа
    Pages/Cabinets/      — кабинеты продавца / производителя / админа
    Pages/...            — главная, корзина, профиль, создание магазина (как было)
    Store/...            — витрина, каталог, карточка товара (подключены к API)
```

## Запуск (разработка)

```bash
npm install
# Бэкенд должен быть запущен (по умолчанию на http://127.0.0.1:8000).
# Vite проксирует /api и /media на бэк (см. vite.config.js), поэтому
# в браузере всё работает с одного origin — как в проде.
npm run dev
```

Переопределить адрес бэка для dev-прокси:
```bash
VITE_DEV_API_TARGET=http://localhost:8000 npm run dev
```

## Сборка (продакшен)

```bash
npm ci
npm run build      # → dist/
```
Содержимое `dist/` отдаётся nginx как статика. Пример конфигурации — в
`../deploy/nginx.conf` (фронт + проксирование /api, /media, /static на gunicorn).

## Переменные окружения (`.env`)

Все опциональны (см. `.env.example`). На одном сервере достаточно значений по
умолчанию.

| Переменная | Назначение | По умолчанию |
|------------|-----------|--------------|
| `VITE_API_BASE` | Базовый путь API | `/api/v1` (тот же домен) |
| `VITE_MEDIA_BASE` | Origin медиа-файлов | пусто (тот же домен) |
| `VITE_DEFAULT_STORE_SLUG` | Магазин для общих маршрутов витрины | пусто → демо-данные |
| `VITE_YANDEX_CLIENT_ID` | OAuth Яндекс (боевой режим) | пусто → sandbox |
| `VITE_VK_CLIENT_ID` | OAuth VK (боевой режим) | пусто → sandbox |
| `VITE_DEV_API_TARGET` | Адрес бэка для dev-прокси | `http://127.0.0.1:8000` |

## Авторизация

- Вход через Яндекс ID / VK ID (PRD 3.2.1). Если client_id не заданы —
  **sandbox-режим**: бэк принимает любой код, вход эмулируется без реальной
  учётки провайдера (удобно для демо).
- Боевой режим: задайте `VITE_YANDEX_CLIENT_ID` / `VITE_VK_CLIENT_ID`. Кнопка
  ведёт на провайдера, который возвращает на `/auth/callback`.
- Новый пользователь → экран выбора роли (`/select-role`): продавец/производитель.
- Покупателю роль присваивается автоматически при первом заказе.
- Админ входит отдельно по email+паролю на `/admin/login`.

JWT хранится в localStorage; access обновляется по refresh автоматически при 401.

## Маршруты

| Путь | Доступ | Назначение |
|------|--------|-----------|
| `/` | все | Главная |
| `/clothingShop-page`, `/clothingShop-page/catalog` | все | Витрина и каталог |
| `/product/:id` | все | Карточка товара |
| `/cart` | все (заказ — после входа) | Корзина + оформление |
| `/login`, `/auth/callback`, `/admin/login` | гость | Вход |
| `/select-role` | вошедший | Выбор роли |
| `/profile` | вошедший | Профиль покупателя |
| `/create-shop`, `/editing-page` | продавец | Создание/настройка магазина |
| `/seller` | продавец | Кабинет продавца |
| `/producer` | производитель | Кабинет производителя |
| `/admin` | админ | Панель администратора |

## Связь с бэкендом

Все вызовы идут через `src/api/*`, который повторяет маршруты бэка под `/api/v1/`.
Сквозной флоу (sandbox): вход → выбор роли → (производитель) товар+фото+публикация
→ (продавец) магазин+оплата+наценка+публикация → (покупатель) витрина→корзина→
оформление (создаётся заказ и платёж, sandbox-оплата подтверждается автоматически)
→ (производитель) сборка/отправка → (админ) подтверждение выплат.

Демо-данные витрины используются только если `VITE_DEFAULT_STORE_SLUG` не задан
или бэк недоступен — чтобы страницы не падали при разработке фронта в одиночку.
